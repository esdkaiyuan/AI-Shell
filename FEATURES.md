# ShellAI 功能完整说明

## 已实现的功能

### 1. AI 助手面板 ✅
**位置**: 终端窗口右侧或底部（可切换）

**核心功能**:
- ✅ 自动获取终端上下文（最近的命令和输出）
- ✅ 手动刷新上下文
- ✅ AI 生成的命令提供三个操作按钮：
  - **插入** - 命令插入到终端输入框
  - **执行** - 直接在终端执行命令
  - **复制** - 复制到剪贴板
- ✅ 面板位置切换（右侧 ⇄ 底部）
- ✅ 自动提取代码块中的命令

**使用方式**:
1. 点击左侧边栏 AI 图标打开/关闭面板
2. 输入问题，AI 会自动读取终端上下文
3. 点击 AUTO/OFF 切换自动上下文模式
4. 点击刷新按钮手动获取上下文

### 2. SSH 连接管理 ✅
**位置**: 顶部工具栏 → 🌐 图标

**核心功能**:
- ✅ 保存 SSH 连接配置（主机、端口、用户名）
- ✅ 支持密码和密钥两种认证方式
- ✅ 管理已保存的连接（编辑、删除）
- ✅ 一键连接到远程服务器
- ✅ SSH 终端与本地终端无缝切换

**使用方式**:
1. 点击工具栏 🌐 图标打开 SSH 管理器
2. 点击"新建连接"填写服务器信息
3. 选择认证方式（密码或密钥）
4. 保存后点击"连接"即可创建 SSH 终端

### 3. 终端搜索 ✅
**位置**: 顶部工具栏 → 🔍 图标

**核心功能**:
- ✅ 实时搜索终端输出
- ✅ 支持区分大小写
- ✅ 支持全字匹配
- ✅ 支持正则表达式
- ✅ 上一个/下一个结果导航
- ✅ 显示匹配数量

**使用方式**:
1. 点击工具栏 🔍 图标或按 Ctrl+F
2. 输入搜索词
3. 使用 Enter/Shift+Enter 或 ↑↓ 按钮导航
4. 按 Esc 关闭搜索

**快捷键**:
- `Enter` - 下一个结果
- `Shift+Enter` - 上一个结果
- `Esc` - 关闭搜索

### 4. 会话管理 ✅
**位置**: 顶部工具栏 → 📁 和 💾 图标

**核心功能**:
- ✅ 保存当前所有终端标签的状态
- ✅ 包括本地终端和 SSH 连接
- ✅ 一键恢复整个工作环境
- ✅ 管理多个会话（加载、删除）
- ✅ 显示会话详情（终端数量、保存时间）

**使用方式**:
1. 点击 💾 图标保存当前会话
2. 输入会话名称（如"开发环境"）
3. 点击 📁 图标查看已保存的会话
4. 点击"加载"恢复会话

**典型场景**:
- 保存"前端开发"会话：本地终端 + 测试服务器 SSH
- 保存"后端调试"会话：多个微服务 SSH 连接
- 每天开始工作时一键加载昨天的环境

### 5. 终端分屏 ✅
**位置**: 顶部工具栏 → ⊞ 图标

**核心功能**:
- ✅ 水平分屏（左右布局）
- ✅ 垂直分屏（上下布局）
- ✅ 每个窗格独立运行终端
- ✅ 支持本地和 SSH 混合分屏
- ✅ 一键开启/关闭分屏

**使用方式**:
1. 在任意终端标签点击 ⊞ 图标
2. 自动创建水平分屏（两个窗格）
3. 再次点击关闭分屏
4. 每个窗格可以独立运行命令

**典型场景**:
- 左侧编辑代码，右侧运行测试
- 上方监控日志，下方执行命令
- 同时查看多个服务器的状态

### 6. 多标签终端 ✅
**位置**: 终端标签栏

**核心功能**:
- ✅ 创建多个终端标签
- ✅ 本地终端和 SSH 终端混合
- ✅ 显示连接类型图标（💻 本地 / 🌐 SSH）
- ✅ 显示主机名（SSH 连接）
- ✅ 关闭单个标签
- ✅ 快速切换标签

**使用方式**:
1. 点击标签栏右侧 + 按钮创建本地终端
2. 通过 SSH 管理器创建远程终端
3. 点击标签切换终端
4. 点击标签上的 × 关闭终端

## 架构设计

### 终端事件总线 (terminalBus)
连接终端和 AI 面板的核心模块：
- 终端注册自己的句柄（insert、run、getContext）
- AI 面板通过总线读取上下文和发送命令
- 自动跟踪活动终端

### 上下文捕获
- 每个终端维护 20KB 滚动缓冲
- 自动去除 ANSI 转义序列
- 只保留纯文本供 AI 理解

### 命令提取
使用正则表达式自动识别：
1. 代码块：```bash ... ```
2. 行内代码：`command`（启发式判断）

## 技术栈

- **前端框架**: React + TypeScript
- **终端模拟**: xterm.js
  - xterm-addon-fit (自适应大小)
  - xterm-addon-web-links (链接识别)
  - xterm-addon-search (搜索功能)
- **进程通信**: Electron IPC
- **状态管理**: React Hooks
- **持久化**: localStorage (会话管理)

## 使用场景示例

### 场景 1: 调试报错
```
终端显示：
$ npm install
npm ERR! code EACCES
npm ERR! syscall access

你在 AI 面板：这个错误怎么解决？
AI：这是权限问题...
    [插入] [执行] [复制]
    sudo chown -R $USER /usr/local/lib/node_modules

点击"执行"→ 命令自动在终端运行
```

### 场景 2: 多服务器运维
```
1. 打开 SSH 管理器
2. 连接到 3 台服务器（web1, web2, db1）
3. 使用分屏同时查看日志
4. 保存为"生产环境监控"会话
5. 明天一键加载整个环境
```

### 场景 3: 前端开发
```
标签 1: 本地终端 - npm run dev
标签 2: 本地终端 - git status
标签 3: SSH 测试服务器 - tail -f logs
标签 4 (分屏):
  - 左: npm test
  - 右: npm run build

AI 面板自动分析构建错误并提供修复建议
```

## 已知限制

1. **搜索功能**: xterm-addon-search 不提供精确的结果计数，当前显示简化版
2. **分屏布局**: 当前仅支持 2 窗格分屏，未来可扩展为多窗格网格布局
3. **AI 上下文**: 限制为最近 20KB 输出，超长日志可能被截断
4. **会话恢复**: 不保存终端历史记录，只恢复连接配置
